# Mini Analytics Challenge

## Lab Goal

Practice the discovery-to-production handoff in one short cycle using cohort datasets.

## Timebox

- Discovery sprint: 20 minutes
- Build sprint: 20 minutes
- QA + share-out: 15 minutes

## Challenge Prompt

Choose one question:

- Which locations show the highest activity over time?
- Where do we see potential capacity pressure?
- Which metric is most stable across days?

## Required Steps

1. Discovery mode

- Ask assistant for 3 ways to answer your chosen question.
- Identify required columns and table grain.
- Record at least 2 data risks.

2. Production mode

- Build one reproducible script or notebook cell sequence.
- Compute at least 2 KPIs.
- Save one output artifact (CSV or chart).

3. Validation

- Check for nulls in required fields.
- Check for duplicate keys at expected grain.
- Check for impossible values (for example, negative counts).

## Deliverable Template

Submit:

- Question
- Data used
- Method summary (3-5 bullets)
- Validation checks performed
- One finding and one limitation

## Reflection Prompts

- What changed between your first idea and final method?
- What did the assistant get wrong or leave out?
- What would you automate before Session 2?
